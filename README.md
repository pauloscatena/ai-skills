# ai-skills

Skills e agentes pessoais para o Claude Code, desenvolvidos e testados com o [skill-creator](https://github.com/anthropics/skills).

## Skills

- **[powerpuff-review](skills/powerpuff-review/SKILL.md)** — Revisão de código feita por três personas com foco próprio (arquitetura, legibilidade/DX, bugs/segurança), inspiradas nas Meninas Superpoderosas. Rode com um pedido do tipo "usa a revisão das Powerpuff Girls nesse arquivo/PR". Testado com evals em `powerpuff-review/evals/evals.json` — versão com skill teve 93,3% de taxa de acerto contra 69,5% da linha de base sem skill.
- **[model-validator](skills/model-validator/SKILL.md)** — Valida modelos de dados relacionais de forma profunda — entidades, relacionamentos, normalização, índices, queries, performance e strategy. Gera relatório detalhado de problemas categorizados por severidade e plano de implementação em 4 fases. Cobertura: 7 dimensões (entidades, multi-tenancy, índices, queries, estratégia, segurança, migrations). Uso: "valida o modelo de dados desse projeto" ou `/model-validator full`.
- **[nosql-validator](skills/nosql-validator/SKILL.md)** — Valida modelos de dados NoSQL — document stores, key-value, column-family, graph databases. Analisa padrão de acesso, denormalização, particionamento, consistência, throughput e custo. Suporta: MongoDB, DynamoDB, Cassandra, Redis, Cosmos DB, Firestore, Neo4j. Uso: "valida o modelo MongoDB desse projeto" ou `/nosql-validator full`.
- **[dotnet-debug-breakpoints](skills/dotnet-debug-breakpoints/SKILL.md)** — Depuração de processos .NET locais com attach, breakpoints e inspeção de estado em tempo de execução. Suporta Visual Studio (EnvDTE), VS Code (netcoredbg) e execução headless. Uso: attach em processo, setar breakpoints, step-through, inspecionar variáveis/call stack.
- **[dotnet-debug-docker](skills/dotnet-debug-docker/SKILL.md)** — Depuração de aplicações .NET rodando em containers Docker e docker-compose. Descobre serviços, coleta logs, faz attach via `vsdbg`, e roda diagnósticos dentro do container. Uso: debug em Docker/docker-compose, attach em container, logs de microserviços.
- **[dotnet-debug-diagnostics](skills/dotnet-debug-diagnostics/SKILL.md)** — Diagnóstico post-mortem de processos .NET: crash dumps (`dotnet-dump`), traces (`dotnet-trace`), counters (`dotnet-counters`) e análise de memória. Uso: crash dump analysis, memory leak, hang/freeze, profiling de performance, análise de GC.
- **[dotnet-debug-fix](skills/dotnet-debug-fix/SKILL.md)** — Analisa causa raiz de bugs .NET e gera plano de correção estruturado, apresentando melhor prática vs. fix mínimo com trade-offs explícitos. Valida a correção com build + testes. Uso: plano de correção, root cause analysis, validação de fix, recomendação de testes de regressão.
- **[qa-unit-tests](skills/qa-unit-tests/SKILL.md)** — Cria, roda e analisa testes unitários isolados. Descobre o framework do projeto (xUnit, NUnit, Jest, pytest, go test), executa a suíte existente, identifica gaps de cobertura e cria testes faltantes seguindo Arrange-Act-Assert. Foco em isolamento (mocks/fakes), boundary cases e naming convencional.
- **[qa-integration-tests](skills/qa-integration-tests/SKILL.md)** — Cria, roda e analisa testes de integração — validação entre módulos, contratos de API, interação com banco real (Testcontainers), handlers + repositórios, transações e filas. Uso: integration tests, testes com banco real, testes endpoint-to-endpoint.
- **[qa-load-tests](skills/qa-load-tests/SKILL.md)** — Cria, roda e analisa testes de carga e stress usando k6. Define cenários (ramp-up, sustain, spike, soak), gera scripts k6, executa contra o alvo e produz relatório com latência p50/p95/p99, throughput e error rate.
- **[qa-e2e-tests](skills/qa-e2e-tests/SKILL.md)** — Executa testes end-to-end, smoke tests e regressão visual em aplicações web usando Chrome MCP (`claude-in-chrome`). Navega na UI real, preenche formulários, clica, valida fluxos completos do usuário e verifica console/rede a cada passo.

### Instalação de skills

Copie a pasta da skill desejada para `~/.claude/skills/<nome-da-skill>/`.

## Agentes

- **[csharp-debugger](agents/csharp-debugger.md)** — Desenvolvedor sênior C#/.NET especializado em depuração real: attach a processo, breakpoints e execução controlada no Visual Studio (via automação COM/EnvDTE) e no VS Code (via `netcoredbg`/`launch.json`), incluindo interceptação de containers Docker/docker-compose (descoberta de serviço→container→PID, logs correlacionados, `docker inspect`/`docker stats`, e attach via `vsdbg` com checagem prévia de viabilidade — `curl`/`wget`/`CAP_SYS_PTRACE`). Ao final sempre entrega um relatório detalhado e um plano de implementação, sem aplicar a correção sozinho. Validado em campo contra um projeto real com stack docker-compose (.NET + Postgres + Ollama).
- **[qa-tester](agents/qa-tester.md)** — Engenheiro de QA sênior que executa validação de verdade em vez de inferir pelo código: roda a suíte de testes automatizada existente como baseline e, em projetos web, navega de verdade via `claude-in-chrome` (formulários, cliques, navegação) checando console e network a cada fluxo. Cobre o roteiro fornecido pelo usuário mais casos de boas práticas de QA (fronteiras, campo vazio, erro/loading). Não corrige código nem invoca outros agentes — sinaliza no relatório quando um achado pede handoff para o `csharp-debugger` (causa raiz em C#/.NET). Relatório final sempre com escopo testado, resultado da suíte automatizada, achados com severidade/repro/evidência, e casos que passaram. Validado em campo contra um projeto real (login, exploração de painel web, achados reais de UX/consistência de dados).
- **[dba](agents/dba.md)** — DBA agent que valida e otimiza bancos de dados relacionais e NoSQL. Audita modelos, índices, queries, performance, segurança e custo. Utiliza as skills `model-validator` (relacional) e `nosql-validator` (NoSQL) para gerar relatórios detalhados e planos de implementação. Detecta automaticamente o engine do banco (via Docker Compose, configs, packages) e carrega a skill adequada. Uso: "valida o banco desse projeto" ou `/dba full`.
- **[devops](agents/devops.md)** — Agente DevOps/SRE sênior que diagnostica problemas de infraestrutura cloud (AWS, Azure, GCP) e servidores via SSH a partir de evidência real — logs, métricas e estado de recursos coletados ao vivo, nunca inferindo causa raiz só pela leitura de IaC parado. Cobre CLI cloud, Kubernetes/Docker, pipelines de CI/CD e acesso SSH direto, sempre avaliando a correção contra os pilares de boas práticas (segurança, confiabilidade, performance, custo, operabilidade). Trata qualquer ação com efeito (reiniciar, aplicar, deletar) como exigindo confirmação explícita antes de executar. Ao final entrega relatório detalhado com evidência coletada e plano de implementação completo com melhor prática vs. fix mínimo, riscos e plano de rollback. Uso: "investiga esse erro em produção" ou "acessa o servidor via SSH e vê o que está travando".
- **[architect](agents/architect.md)** — Arquiteto de soluções especializado na categoria Couplers de Code Smells: Feature Envy, Inappropriate Intimacy, Message Chains, Middle Man e Incomplete Library Class. Localiza candidatos por padrão de código (grep), confirma cada achado em contexto para evitar falso positivo, e aplica a Lei de Deméter e o "Teste do Botão" (ortogonalidade) para avaliar acoplamento real. Também cobre desvinculação física — dependências cíclicas entre arquivos/pastas/projetos que estendem build e teste. Entrega relatório por heurística com localização exata e plano de refatoração priorizado (técnica, passos, risco e validação), sem aplicar mudança estrutural sem aprovação. Uso: "revisa o acoplamento desse módulo" ou "acha as message chains nesse projeto".
- **[precatorio-municipal](agents/precatorio-municipal.md)** — Especialista jurídico em precatórios de esfera Municipal (PMSP, PCRJ e demais municípios). Revisa specs técnicas quanto a completude jurídica, avalia fluxos/telas como um operador real e aponta quais campos de retorno de scraper/crawler são juridicamente relevantes para essa esfera.
- **[precatorio-estadual](agents/precatorio-estadual.md)** — Especialista jurídico em precatórios de esfera Estadual (TJSP, TJDFT). Mesma cobertura do agente municipal, aplicada às particularidades estaduais.
- **[precatorio-federal](agents/precatorio-federal.md)** — Especialista jurídico em precatórios de esfera Federal (TRF1 a TRF8, PJe). Mesma cobertura do agente municipal, aplicada às particularidades federais.
- **[precatorio-trabalhista](agents/precatorio-trabalhista.md)** — Especialista jurídico em precatórios de esfera Trabalhista (TRT2, TRT15, PJe trabalhista). Mesma cobertura do agente municipal, aplicada às particularidades trabalhistas.

### Instalação de agentes

Copie o arquivo do agente desejado para `~/.claude/agents/<nome-do-agente>.md` — fica disponível automaticamente em qualquer projeto (escopo de usuário, não por repositório).
